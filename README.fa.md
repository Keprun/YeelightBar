# YeelightBar

[English](README.md) · [Русский](README.ru.md) · [中文](README.zh-CN.md) · **فارسی** · [Español](README.es.md) · [العربية](README.ar.md)

یک اپلیکیشن بومی macOS برای کنترل دستگاه‌های شبکه‌ای **Yeelight** — ساخته‌شده برای **Yeelight Screen Light Bar Pro**
(`YLTD003`)، اما نوارها و لامپ‌های RGB ساده را هم به‌خوبی هدایت می‌کند. نرم‌افزار رسمی فقط برای Windows است؛ این یک جایگزین تمیز و
سریع مبتنی بر SwiftUI است با همگام‌سازی نور محیطی با صفحه (ambilight)، واکنش‌پذیری به موسیقی و کنترل گروهی چند لامپ.

> نه ابری، نه حسابی. همه‌چیز روی شبکه‌ی محلی (LAN) شما اتفاق می‌افتد.

## امکانات

- **کنترل کامل** — روشن/خاموش، سفید جلویی (روشنایی + دمای رنگ)، RGB محیطی، صحنه‌های آماده.
- **کنترل گروهی / «mix»** — چند لامپ را انتخاب کنید و آن‌ها را با هم هدایت کنید؛ هر لامپ گویش پروتکلی خودش را صحبت می‌کند
  (نوار یک کانال محیطی جداگانه‌ی `bg` دارد، اما نوارهای ساده ندارند).
- **همگام‌سازی نور محیطی با صفحه (ambilight)** — ScreenCaptureKit از صفحه‌ی شما نمونه‌برداری می‌کند و رنگ را به کانال محیطی لامپ
  با نرخ حدود ۲۰ هرتز روی یک نشست UDP استریم می‌کند.
  - **نمایشگر + ناحیه به‌ازای هر لامپ**: در یک چیدمان چندمانیتوره، هر لامپ می‌تواند از یک نمایشگر *متفاوت* و یک
    ناحیه‌ی *متفاوت* از آن (بالا / پایین / چپ / راست / کامل) نمونه‌برداری کند — مثلاً نوار از بالای صفحه‌ی اصلی شما می‌گیرد
    در حالی که یک نوار زیرمیزی از پایین یک نمایشگر دیگر نمونه می‌گیرد.
  - نمونه‌برداری مستقل از وضوح (روی نسبت‌های ۱۶:۹، ۴K، عمودی و فوق‌عریض‌های ۳۲:۹ یکسان کار می‌کند).
  - پیش‌نمایش زنده: یک پنل به‌شکل صفحه برای هر نمایشگر که دقیقاً نشان می‌دهد هر لامپ از کدام ناحیه نمونه می‌گیرد، با رنگ زنده‌ی آن.
- **واکنش‌پذیری به موسیقی** — صدای *سیستم* را ضبط می‌کند (بدون میکروفون) و آن را با فیلترهای IIR به بم/میانه/زیر تقسیم می‌کند.
  - حالت **Beat** روشنایی را روی ضربه‌ی کیک پمپ می‌کند؛ حالت **Spectrum** بم را به قرمز / میانه را به سبز / زیر را به آبی نگاشت می‌کند.
- **دو سطح کاربری** — یک پنل فشرده در نوار منو برای تنظیمات سریع و یک پنجره‌ی کامل و قابل‌تغییراندازه (`NavigationSplitView`)
  برای راه‌اندازی.
- **پایدار روی یک شبکه‌ی واقعی** — کشف خودکار (SSDP + اسکن فعال زیرشبکه)، اتصال مجدد هنگام تغییر IP در DHCP، و
  کنترل سریالی‌شده تا لامپ هرگز فرمانی را از اتصالات هم‌زمان از دست ندهد.

## پیش‌نیازها

- macOS 13 (Ventura) یا جدیدتر، Apple Silicon یا Intel.
- دستگاه(های) Yeelight با **LAN Control** فعال (اپلیکیشن Yeelight ← دستگاه ← *LAN Control*).
- مجوز **Screen Recording** (System Settings ← Privacy & Security) برای حالت‌های همگام‌سازی با صفحه و موسیقی.

## ساخت و اجرا

### Xcode
`YeelightBar.xcodeproj` را باز کنید و scheme مربوط به **YeelightBar** را اجرا کنید (⌘R). پروژه از `project.yml`
با [XcodeGen](https://github.com/yonaskolb/XcodeGen) تولید می‌شود؛ پس از ویرایش مشخصات، `xcodegen generate` را اجرا کنید.

### Swift Package Manager (بدون نیاز به Xcode)
```sh
swift build
./scripts/bundle.sh          # ساخت + امضای build/YeelightBar.app
open build/YeelightBar.app
```
`scripts/setup-signing.sh` یک هویت پایدار و خودامضای code-signing می‌سازد تا مجوز Screen-Recording پس از
بازسازی‌ها باقی بماند (یک امضای ad-hoc در هر بیلد تغییر می‌کند و موجب نمایش دوباره‌ی درخواست مجوز می‌شود).

## `yeectl` — ابزار خط فرمان

یک CLI کوچک برای تست و اسکریپت‌نویسی پروتکل:

```sh
swift run yeectl discover                 # SSDP
swift run yeectl auto                      # SSDP، در صورت ناکامی بازگشت به اسکن فعال زیرشبکه
swift run yeectl state   <ip>
swift run yeectl on|off  <ip>
swift run yeectl bright  <ip> <0-100>
swift run yeectl ct      <ip> <1700-6500>
swift run yeectl rgb     <ip> <hex e.g. FF8800>   # کانال محیطی / bg
swift run yeectl rainbow <ip> [seconds]           # تست استریم UDP با نرخ ۲۰ هرتز
```

## معماری

```
Sources/
  YeelightKit/            # کتابخانه‌ی فقط-انتقال، بدون UI
    Yeelight.swift        # کنترل JSON روی TCP 55443 + نشست استریم UDP 55444
    Discovery.swift       # کشف SSDP از طریق multicast
    Scan.swift            # اسکن فعال زیرشبکه + اعتبارسنجی IP دستی
  yeectl/                 # CLI
  YeelightBarApp/         # اپلیکیشن SwiftUI
    LampController.swift   # store با @MainActor: کشف، کنترل گروهی، هماهنگی همگام‌سازی
    ScreenSyncEngine.swift # ضبط چندنمایشگری ← رنگ به‌ازای هر (نمایشگر، ناحیه) ← توزیع روی UDP
    MusicSyncEngine.swift  # ضبط صدای سیستم ← beat/spectrum ← توزیع روی UDP
    FullView.swift / MenuPanelView.swift
```

پروتکل شبکه‌ی محلی Yeelight (کنترل TCP، دست‌دهی استریم UDP، و کانال‌های عجیب `main_power`/`bg_power` نوار)
در [`PROTOCOL.md`](PROTOCOL.md) مستند شده است.

## نکاتی درباره‌ی Screen Light Bar Pro

این لامپ دو کانال مستقل دارد — سفید جلویی (`set_power` / `main_power`) و RGB محیطی
(`bg_set_power` / `bg_set_rgb`) — بنابراین می‌توانید حالت «فقط محیطی» را اجرا کنید. ویژگی `power` آن غیرقابل‌اتکا است (حتی
وقتی جلوی لامپ تاریک است روی `on` گیر می‌کند)؛ به‌جای آن، اپلیکیشن `main_power` را می‌خواند. نوارهای ساده یک کانال دارند و
`dev_toggle` مخصوص نوار را رد می‌کنند، پس کنترل بر اساس نوع دستگاه ارسال می‌شود.
